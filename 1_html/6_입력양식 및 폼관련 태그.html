폼 관련 태그

form태그 내의 submit버튼 클릭시 해당 form 내에 작성된 사용자가 입력한 값들을 서버로 넘기면서 요청(request) 하는 역할을 수행한다.

[표현법]
form action="서버주소" method="GET/POST"
* action 속성 : form태그 내부의 입력한 값들을 전송할 서버의 주소
* method 속성 : 서버로 데이터 요청시 전송방식을 지정하는 속성
1. GET : 요청시 사용자가 전달하는 값들을 URL에 노출하여 보내는 방식, 기본 형식이라 생략가능
2. POST : 요청시 사용자가 전달한 값들을 URL BODY 영역에 담아 전송하는 방식

라디오버튼과 체크박스

라디오 버튼

-제시한 여러개의 값들 중 오직 "1개만" 선택해야 하는 경우 사용
-서버로 요청시 선택된(checked) 라디오 버튼의 value 값이 전송된다.
-name 속성을 통해 radio 버튼을 그룹으로 만든다

체크박스

-제시한 여러개의 값들 중 여러개를 선택해야 하는 경우 사용
-현재 선택된 체그 박스의 value 값들이 서버로 전송된다.
-name 속성값을 통일시켜 하나의 그룹으로 만들어 줘야 한다.

토핑 :

select 태그와 option 태그

- 사용자에게 직접 키보드로 값을 입력받는게 아니라, 내가 제시한 여러 옵션들 중 하나를 선택할 수 있게 하는 태그
- 서버로 요청시 현재 선택된(selected) 값이 서버로 넘어 간다
- 각 option태그에 value 속성값을 작성한 경우, value 값이 서버로 넘어가고, 명시하지 않는 경우 content 영역의 값이 서버로 전송된다.
